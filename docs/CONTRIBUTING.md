# Contributing

## Git Workflow

Main Branch:

main

Development Branch:

develop

Feature Branch Example:

feature/customer-menu

feature/admin-dashboard

feature/kitchen-orders

---

## Commit Messages

Examples:

feat: add customer menu page

fix: resolve order status bug

refactor: improve cart store logic

docs: update API contracts

---

## Pull Requests

Rules:

* One feature per pull request
* Update documentation when necessary
* Test before merging
* Keep commits meaningful

---

## Code Standards

Frontend:

* Use functional components
* Use TypeScript
* Use Tailwind CSS
* Keep components reusable

Backend:

* Follow route → controller → service pattern
* Use TypeScript
* Validate inputs
* Keep business logic inside services
